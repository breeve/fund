# 深圳房产分析 - 地理层级划分

## 目标

从真实购房需求出发，支持1-5年跟踪

## 当前方案

```
行政区 → 板块 → 小区
```

## 问题分析

| 问题               | 说明                               |
| ------------------ | ---------------------------------- |
| 板块边界模糊       | 同一小区可能同时属于多个板块认知   |
| 行政边界 vs 生活圈 | 按街道办划分 vs 按15分钟生活圈划分 |
| 购房决策路径       | 实际上购房者先锁定片区，再选小区   |

## 建议方案

### 四级结构

| 层级 | 名称   | 示例                           | 说明                   |
| ---- | ------ | ------------------------------ | ---------------------- |
| 1    | 行政区 | 南山区、宝安区                 | 预算筛选               |
| 2    | 板块   | 科技园、碧海、红山             | 通勤+配套核心决策      |
| 3    | 生活圈 | 深大花园周边、碧海湾地铁站周边 | 步行可达范围，共享配套 |
| 4    | 小区   | 具体楼盘                       | 最终决策               |

### 划分原则

- **行政区**：按政府划分，无需讨论
- **板块**：市场通用概念，按地铁/配套形成的区域认知（如"碧海"）
- **生活圈**：地铁站周边500-800米步行范围，共享同一套配套
- **小区**：具体楼盘

### 板块需关联的属性

1. **通勤关联**
   - 目标通勤方向
   - 通勤时间/方式
   - 换乘次数

2. **家庭阶段**
   - 有娃：学区权重最高
   - 无娃：交通+增值潜力
   - 养老：医疗+环境
   - 当前阶段 → 下一阶段预期
