# 生活圈房产关注属性及合法数据获取渠道

> 本文档梳理深圳购房决策中"生活圈"级别需要关注的核心属性，以及各属性合法合规的 API / 数据获取渠道，评估自动化可行性。
>
> 生活圈定义：以地铁站为中心，500-800 米步行可达范围，居民共享同一套日常配套的区域。是"板块"下更细的粒度，是购房者在锁定板块后实际选房的决策单位。

---

## 一、属性分类

### 1. 锚点与边界

| 属性 | 说明 |
|------|------|
| 锚定地铁站 | 生活圈内最近的地铁站（可能多个） |
| 站间距 | 与相邻地铁站的距离（判断线路密度） |
| 步行覆盖半径 | 以地铁站为中心的步行等时圈范围 |
| 地理边界 | 等时圈 polygon 或500-800m缓冲圈 |
| 所属板块/行政区 | 上级归属 |

### 2. 地铁服务能力

| 属性 | 说明 |
|------|------|
| 线路数 | 经过的地铁线路数量 |
| 早高峰发车间隔 | 反映运力紧张程度 |
| 末班车时间 | 晚归人群关注 |
| 换乘次数 | 从该站到主要目的地的换乘情况 |
| 日均客流量 | 反映站点繁忙程度（深圳地铁官方有公开） |
| 站内设施 | 洗手间、无障碍电梯、换乘通道 |

### 3. 日常通勤指标

| 属性 | 说明 |
|------|------|
| 到主要目的地的耗时 | 到福田、南山科技园、前海等地的地铁+步行时间 |
| 高峰期拥挤度 | 地铁车厢拥挤指数 |
| 首末班车时间 | 该站的线路首末班时间 |
| 通勤总换乘次数 | 路径规划结果 |

### 4. 教育配套

| 属性 | 说明 |
|------|------|
| 幼儿园数量/步行距离 | 最近幼儿园的名称和步行分钟数 |
| 小学数量/步行距离 | 最近小学的名称和步行分钟数 |
| 中学数量/步行距离 | 最近中学的名称和步行分钟数 |

### 5. 商业与生活服务

| 餐饮门店密度 | 反映居住氛围成熟度 |
| 健身房/游泳池 | 运动设施（改善型需求） |
| 银行/快递驿站 | 基础服务设施覆盖 |

### 6. 医疗健康

| 属性 | 说明 |
|------|------|

### 7. 环境与安全

|------|------|

## 二、合法合规的数据获取渠道

### A. 地图开放平台（核心渠道，覆盖 80% 属性）

适合：POI 查询、距离计算、路径规划、等时圈、地理边界

| 平台 | 地址 | 核心能力 |
|------|------|----------|
| 高德开放平台 | https://lbs.amap.com/ | POI 查询、路径规划、距离测量、行政区边界 |
| 腾讯位置服务 | https://lbs.qq.com/ | POI 检索、距离/时间批量计算 |
| 百度地图开放平台 | https://lbsyun.baidu.com/ | 同上，地理分析能力强 |

**生活圈专项自动化要点**：

1. **等时圈计算**
   - 高德路径规划 API 支持 `strategy=walking`（步行）
   - 以地铁站为起点，设置不同时间阈值（如10分钟、15分钟、20分钟）
   - 反向绘制等时圈 polygon，即为生活圈边界
   - 注：高德付费版提供等时圈服务，免费版可用多方向路径采样近似

2. **POI 批量查询**
   - 以地铁站坐标为中心，按分类代码批量查询周边 POI
   - 设置合理半径（800m），避免跨生活圈污染
   - 建议按以下分类分批查询：餐饮（050000）、购物（060000）、医疗（090000）、教育（150300）、交通（150000）、金融（160000）、公园（130100）

3. **通勤时间批量计算**
   - 预设多个固定目的地（如福田站、南山科技园站、前海站）
   - 批量调用路径规划 API 获取耗时
   - 作为生活圈"通勤价值"的量化核心指标

**合规性**：需 Key 可商业使用，免费档位覆盖中小规模调用。

## 四、建议方案

### 核心原则

生活圈是购房决策的**最后一公里**，数据以地铁站为锚点向外辐射采集，重点关注**步行可达性**和**日常高频配套密度**。相比板块，生活圈更强调微观尺度的体验一致性。

### 自动化管线建议

#### 第一梯队（高优全自动，每日/每周更新）

1. **地铁站 POI 锚点建设**
   - 以高德 POI API 批量获取深圳所有地铁站坐标
   - 按线路/站点维度存储，作为所有生活圈的锚点基础表

2. **等时圈边界计算**
   - 以每个地铁站为中心，调用高德步行路径规划 API
   - 采样8-16个方向，计算10分钟、15分钟、20分钟可达范围
   - 生成 polygon 作为生活圈边界

3. **周边 POI 全量采集**
   - 以地铁站坐标为圆心，800m 为半径
   - 按分类批量查询（餐饮/购物/医疗/教育/公园/交通）
   - 按生活圈聚合统计 POI 密度和覆盖度

4. **固定通勤方向耗时**
   - 预设目的地池：福田CBD、南山科技园、前海、蛇口、深圳北站等
   - 批量计算各生活圈到各目的地的地铁+步行总耗时
   - 作为生活圈通勤价值核心指标

5. **地铁服务能力采集**
   - 爬取深圳地铁官网各站点首末班车时间
   - 按线路整理，作为生活圈地铁便利性的基础数据

#### 第二梯队（半自动，月度更新）

6. **租金行情**
   - 接入贝壳或克尔瑞数据接口（付费）
   - 或定期爬取 58同城/安居客租房数据（注意反爬）

7. **小区基础数据**
   - 从贝壳/克尔瑞获取小区楼龄、物业、车位比等信息
   - 按生活圈聚合

#### 第三梯队（人工为主，季度更新）

8. **不利因素标注**
   - 维护一套生活圈级的不利因素清单（垃圾站、变电站等）
   - 来自政府公示 + 用户反馈

9. **内涝黑点**
   - 每年雨季前更新一次水务局公示的内涝黑点

10. **居住人群画像**
    - 结合长租公寓品牌分布和租金水平，侧面推断
    - 人工整理为主
