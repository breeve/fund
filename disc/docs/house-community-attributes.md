# 小区房产关注属性及合法数据获取渠道

> 本文档梳理深圳购房决策中"小区"（具体楼盘）级别需要关注的核心属性，以及各属性合法合规的 API / 数据获取渠道，评估自动化可行性。
>
> 小区定义：购房的最终决策单位，即具体楼盘/小区，如"深大花园""碧海湾一期"。是四个层级中粒度最细、信息密度最高的层级。

---

## 一、属性分类

### 1. 基础档案

| 属性 | 说明 |
|------|------|
| 小区名称 | 推广名 + 备案名（可能不同） |
| 所在行政区 | 南山、宝安等 |
| 所在板块 | 市场通用板块名称 |
| 所在生活圈 | 锚定地铁站名称 |
| 详细地址 | 街道+社区+路名+门牌号 |
| 竣工年份/楼龄 | 首次交付时间，推算楼龄 |
| 总栋数/总户数 | 规模体量 |
| 开发商 | 知名开发商溢价更高 |
| 物业公司 | 品牌物业通常更保值 |
| 物业费 | 元/㎡·月 |
| 小区定位 | 刚需/改善/豪宅/安居房等 |

### 2. 建筑信息

| 属性 | 说明 |
|------|------|
| 建筑面积 | 总占地面积 |
| 建筑面积 | 总建筑面积 |
| 容积率 | 容积率越高越拥挤 |
| 绿化率 | 绿化覆盖率 (%) |
| 总栋数 | 楼栋数量 |
| 层数/总层高 | 楼层高低影响采光和视野 |
| 梯户比 | 每部电梯服务多少户（影响等电梯体验） |
| 得房率 | 套内面积/建筑面积 (%) |
| 朝向 | 南向/北向/东西向分布比例 |
| 层高 | 标准层高（米） |
| 外立面材料 | 涂料/瓷砖/石材/铝板（影响外观耐久度） |
| 建筑结构 | 框架/框剪/钢构 |

### 3. 停车位与交通

| 属性 | 说明 |
|------|------|
| 车位数 | 总车位数量 |
| 车户比 | 车位:户数（如 1:1.5） |
| 地上/地下车位数 | 车位分布 |
| 停车费 | 月租/临停费用 |
| 车位可售/可租 | 是否有产权车位出售 |
| 最近地铁站 | 名称和步行距离（米/分钟） |
| 地铁线路 | 经过的线路 |
| 公交线路数 | 周边公交线路数量 |
| 高速公路/快速路出入口 | 距离和可达性 |
| 机场/火车站可达性 | 距离主要交通枢纽的耗时 |
| 高铁站可达性 | 距深圳北站/福田站/坪山站等高铁站的公共交通耗时 |
| 非机动车道情况 | 小区周边是否有独立非机动车道，骑行安全性 |
| 高峰期拥堵指数 | 周边道路早/晚高峰拥堵程度（来自高德交通态势 API） |

### 4. 教育配套

| 属性 | 说明 |
|------|------|
| 对口幼儿园 | 名称和距离 |
| 对口小学 | 名称和距离 |
| 对口初中 | 名称和距离 |
| 对口高中 | 高中通常按中考分数派位（非对口，这里指最近的重点高中） |
| 学校评级 | 省一级/市一级/区一级 |
| 是否有学位预警 | 学位是否紧张 |
| 是否有九年一贯制 | 学校类型 |
| 是否有私立/国际部 | 升学路径选择 |

### 5. 商业与生活配套

| 属性 | 说明 |
|------|------|
| 小区底商 | 商铺业态和经营状况 |
| 最近菜市场/生鲜 | 步行距离 |
| 最近超市/便利店 | 步行距离 |
| 最近大型商场 | 步行距离 |
| 最近餐饮一条街 | 步行距离 |
| 最近银行 | 步行距离 |
| 快递驿站/蜂巢 | 位置 |
| 美容店密度 | 周边美容/美发/SPA 门店数量（反映居住人群结构和消费层次） |
| 儿童游乐设施 | 小区内部儿童滑梯/游乐场，或周边1km内儿童乐园（反映亲子氛围） |
| 周边不利因素 | 垃圾站、变电站、高架桥、墓地等 |

### 6. 医疗健康

| 属性 | 说明 |
|------|------|
| 最近社康中心 | 步行距离 |
| 最近三甲医院 | 公共交通耗时 |
| 最近私立医院/诊所 | 步行距离（口腔、儿科等） |
| 药店密度 | 周边药店数量 |

### 7. 环境与安全

| 属性 | 说明 |
|------|------|
| 内部园林景观 | 绿化维护情况 |
| 噪音源 | 是否有高速/铁路/学校广播噪音 |
| 采光遮挡 | 楼间距和朝向 |
| 物业管理水平 | 门禁、巡逻、清洁维护 |
| 安防系统 | 监控覆盖率 |
| 人车分流 | 是否完全人车分流 |
| 电梯品牌/维保 | 电梯安全和维护状况 |
| 消防设施 | 消防栓、灭火器年检情况 |
| 内涝风险 | 小区是否在历史内涝黑点上 |

### 8. 户型产品

| 属性 | 说明 |
|------|------|
| 户型种类 | 一室/两室/三室/四室/复式/别墅等 |
| 主力户型 | 占比最大的户型 |
| 户型面积段 | 最小/最大/主力面积 |
| 朝向分布 | 各朝向占比 |
| 通风采光 | 户型设计优劣 |
| 得房率 | 分户型得房率 |
| 装修状况 | 毛坯/精装/豪装分布 |
| 是否有赠送面积 | 阳台/飘窗/入户花园等 |

### 9. 价格与行情

| 属性 | 说明 |
|------|------|
| 当前挂牌价 | 元/㎡ 或 万元/套 |
| 挂牌价历史走势 | 近6个月/1年/3年走势 |
| 近期成交价 | 最近3-6个月的成交均价 |
| 成交周期 | 从挂牌到成交的平均天数 |
| 议价空间 | 挂牌价与成交价的差额比例 |
| 在售套数 | 当前挂牌数量 |
| 在租套数 | 当前挂牌出租数量 |
| 租金水平 | 元/㎡·月，分户型 |

### 10. 小区口碑与舆情

| 属性 | 说明 |
|------|------|
| 业主论坛/社群口碑 | 业主对物业/质量的评价 |
| 历史维权事件 | 是否有过大规模维权（质量/物业/规划变更） |
| 链家/贝壳评分 | 平台用户评分 |
| 媒体相关报道 | 负面新闻（质量事故、虚假宣传等） |
| 规划变更风险 | 周边是否有新建高架/垃圾站等不利规划 |

## 二、合法合规的数据获取渠道

### A. 地图开放平台（覆盖地理+配套+交通）

适合：地址坐标、到地铁站距离、周边 POI、路径规划

| 平台 | 地址 | 可获取数据 |
|------|------|------------|
| 高德开放平台 | https://lbs.amap.com/ | 地理编码（地址→坐标）、逆地理编码（坐标→小区信息）、POI 查询、距离测量 |
| 腾讯位置服务 | https://lbs.qq.com/ | 同上，距离计算准确 |
| 百度地图开放平台 | https://lbsyun.baidu.com/ | 同上，数据覆盖全面 |

**小区专项自动化要点**：
1. **坐标获取**：通过高德地理编码 API，输入小区名称+深圳市，获取精确坐标
2. **周边配套分析**：以小区坐标为中心，设置不同半径（300m/500m/1000m），按分类批量查询 POI
3. **地铁距离**：调用距离测量 API，计算直线距离和步行距离
4. **不利因素检测**：以小区坐标为中心，在500m内搜索 `"垃圾中转站" "变电站" "高压线路" "墓地"` 等关键词

**合规性**：需 Key，免费档位覆盖中小规模调用。

---

### B. 贝壳/链家（覆盖房价行情+基础档案）

适合：挂牌价、成交价、历史走势、在售套数、小区基础信息

| 平台 | 说明 | 可获取数据 |
|------|------|------------|
| 贝壳找房 | 深圳最大二手房平台 | 小区详情页：楼龄、户数、物业费、在售、在租、成交历史、均价走势 |
| 链家网 | 类似贝壳 | 同上 |
| 中原地产 | 老牌中介 | 成交行情，部分小区有历史成交记录 |

**自动化要点**：
- 贝壳/链家**无公开免费 API**，需商务合作
- 小区详情页可爬取（需处理反爬），静态信息（楼龄/户数等）稳定
- 动态价格数据建议通过商务渠道获取，爬取存在法律风险

**合规性**：有版权的房源数据（图片、文字描述）不得直接使用或二次传播。成交数据建议通过正规商务渠道。

---

### C. 政府公开数据（覆盖教育+规划+产权）

适合：学区划分、产权信息、预售许可、小区竣工备案

| 来源 | 说明 |
|------|------|
| 深圳教育局官网 | 学区划分方案（PDF，按小区名称检索） |
| 深圳规划和自然资源局 | 竣工验收备案信息，小区建筑面积等 |
| 深圳住建局 | 预售许可证信息（包含小区名称、楼栋、面积） |
| 深圳不动产登记中心 | 产权年限、抵押查封情况（部分公示） |
| 市场监管局 | 开发商企业信用信息 |

**自动化要点**：
- 学区划分：教育局按街道/社区发布，可按小区名称检索 PDF
- 住建局预售许可证查询：官网有查询入口，可按小区名检索
- 产权信息：部分公示，不动产登记中心可查抵押查封状态

**合规性**：政府公开信息，免费使用。

---

### D. 克尔瑞/中指研究院（覆盖市场数据）

适合：成交走势、板块排名、竞品分析

| 平台 | 说明 |
|------|------|
| 克尔瑞（CRIC） | 付费订阅，提供小区级/户型级成交数据 |
| 中指研究院 | 付费月报，含市场整体和重点小区数据 |
| 贝壳研究院 | 部分报告免费公开，含小区分析 |

**合规性**：付费数据，严格按协议使用。

---

### E. 地理信息公共服务平台（覆盖边界+不利因素）

适合：小区边界、楼栋分布、不利因素标注

| 平台 | 地址 | 可获取数据 |
|------|------|------------|
| 全国地理信息资源目录服务系统 | https://www.webmap.cn/ | 建筑轮廓 shp，辅助判断楼栋分布和间距 |
| 高德/百度 POI API | 地图开放平台 | 不利因素（垃圾站/变电站/墓地/高架）位置和距离 |
| 深圳水务局公示 | 政府网站 | 内涝黑点分布 |

**自动化要点**：
- 不利因素检测：调用高德 POI 关键词搜索，在小区周边500-1000m内检索禁忌类 POI
- 内涝黑点：水务局每年雨季前公示一次，可人工整理后标注

---

### F. 用户评价与舆情（覆盖口碑）

适合：业主口碑、维权记录、媒体曝光

| 来源 | 说明 |
|------|------|
| 贝壳/链家小区详情页 | 用户评分和标签（如"近地铁""环境好"） |
| 业主论坛（本地宝、房天下业主论坛） | 业主真实评价和维权信息 |
| 天眼查/企查查 | 开发商法律诉讼、物业公司投诉记录 |
| 深圳阳光政务平台 | 投诉和答复记录 |
| 各大房产自媒体 | 小区专题分析 |

**自动化要点**：
- 用户评分：贝壳/链家有评分，可通过小区详情页获取
- 维权事件：天眼查可查企业法律诉讼，按小区名搜索开发商
- 舆情：定期抓取房产自媒体文章，按小区名聚合

## 三、自动化可行性评估

| 数据类型 | 推荐来源 | 可自动化 | 备注 |
|----------|----------|----------|------|
| 小区基础档案（名称/地址/楼龄/户数） | 贝壳/克尔瑞 | ✅ 高 | 小区详情页可爬，静态数据 |
| 建筑面积/容积率/绿化率 | 贝壳/住建局预售许可 | ✅ 高 | 数据稳定，变化少 |
| 物业费/物业公司/梯户比 | 贝壳小区详情 | ✅ 高 | 静态数据 |
| 户型种类/面积段 | 贝壳/克尔瑞 | ✅ 高 | 按小区批量查询 |
| 当前挂牌价/成交价 | 贝壳/克尔瑞 | ⚠️ 中 | 需商务合作或爬取（有风险） |
| 历史价格走势 | 克尔瑞/贝壳 | ⚠️ 中 | 需商务合作 |
| 在售/在租套数 | 贝壳 | ⚠️ 中 | 小区在售列表页可爬 |
| 租金水平 | 58同城/贝壳 | ⚠️ 中 | 有反爬 |
| 对口学校/学区 | 教育局 PDF | ❌ 低 | PDF 不结构化，人工维护 |
| 车位数量/车户比 | 贝壳小区详情 | ✅ 高 | 静态数据 |
| 最近地铁站/距离 | 高德 POI API | ✅ 高 | 批量计算 |
| 周边 POI 密度 | 高德 POI API | ✅ 高 | 半径查询统计 |
| 不利因素检测 | 高德 POI + 政府公示 | ✅ 高 | 500m内禁忌 POI 检索 |
| 内涝风险 | 水务局公示 | ⚠️ 低 | PDF 为主，人工维护 |
| 业主口碑/评分 | 贝壳/链家 | ⚠️ 中 | 小区详情页可爬 |
| 维权历史/法律诉讼 | 天眼查 | ✅ 高 | 按开发商名批量查询 |
| 媒体舆情 | 搜索 + 自媒体 | ⚠️ 中 | 定期搜索，按小区名聚合 |

## 四、建议方案

### 核心原则

小区是购房决策的**最终落点**，数据以**小区档案为核心**，外接周边配套（来自高德）和市场行情（来自贝壳/克尔瑞）。重点关注：
- **基础档案完整性**：楼龄、户型、物业等核心静态信息
- **价格行情现势性**：挂牌价、成交价、议价空间
- **不利因素排查**：通过 POI 检索排除硬伤

### 自动化管线建议

#### 第一梯队（高优全自动，一次建库+定期更新）

1. **小区基础档案库**
   - 从贝壳/克尔瑞批量抓取小区详情页（楼龄/户数/物业/车位/户型等）
   - 初始建库后，每季度增量更新
   - 关键字段：小区名称、竣工年份、总户数、物业公司、物业费、车位数、容积率、绿化率

2. **小区坐标与周边 POI 分析**
   - 通过高德地理编码 API 获取每个小区的精确坐标
   - 以小区坐标为中心，500m/1000m 两个半径查询 POI 密度
   - 分类统计：地铁站、学校、医院、商场、公园、便利店、餐厅
   - 检索禁忌 POI：垃圾站、变电站、高架桥、墓地等

3. **地铁距离计算**
   - 以小区坐标调用高德距离测量 API
   - 计算到最近地铁站的直线距离和步行距离
   - 作为小区"地铁便利性"核心指标

4. **学区基础数据**
   - 从教育局官网爬取学校列表（含地址、评级）
   - 通过高德坐标匹配，建立 学校→地址 映射
   - 小区→学校距离由坐标计算得出（精确到步行距离）

5. **维权历史扫描**
   - 通过天眼查/企查查，按开发商名称搜索法律诉讼记录
   - 筛选涉及"房屋质量""虚假宣传""物业纠纷"等关键词的案件

#### 第二梯队（半自动，月度更新）

6. **价格行情监控**
   - 接入贝壳或克尔瑞数据接口
   - 按小区跟踪：挂牌价、成交价、在售套数变化
   - 设置价格波动告警（超过一定幅度自动提醒）

7. **在租行情**
   - 定期抓取贝壳/58同城租房数据
   - 按小区聚合租金走势

8. **舆情监控**
   - 定期按小区名搜索新闻和自媒体文章
   - 负面报道人工核实后入库

#### 第三梯队（人工为主，关键节点更新）

9. **学区划分映射**
   - 每年4-5月教育局发布后，人工更新 学区-小区 映射表
   - 这是购房最关键但最难结构化的数据，建议维护 Excel 手工表

10. **不利因素动态标注**
    - 用户反馈 + 人工巡查
    - 重大不利因素（新建垃圾站、高架）由业主报料后录入

## 五、附：小区不利因素 POI 关键词清单

| 类型 | 高德搜索关键词 |
|------|---------------|
| 垃圾站/垃圾处理 | 垃圾中转站、垃圾处理厂、垃圾填埋场 |
| 变电站/高压线 | 变电站、高压线路、供电设施 |
| 墓地/殡仪馆 | 墓地、殡仪馆、火葬场 |
| 工厂/仓储 | 工厂、仓储中心、加油站（敏感人群） |
| 高架/铁路 | 高架桥、铁路、地铁维修段 |
| 污水处理厂 | 污水处理厂、泵站 |
