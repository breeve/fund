# 服务端

## 数据组织方式

1. 使用多个 YAML 文件组织不同层级的数据结构和组成信息，作用类似 DB 设计中的 schema
2. 使用 YAML 文件描述不同数据信息的自动收集方式、收集频率等属性，不与数据信息的组织和组成关系混在一起
3. 对于部分数据信息，提供手动标注的方式（具体哪些需要手动标注，后续再定，当前仅在 YAML 组织形态上预留设计）

## 数据存储

1. 选取 PostgreSQL 作为主要数据存储库
2. 支持非数据库存储（如文件存储）
3. 支持同步文件存储到云端（OSS/S3）

## API

1. 提供 RESTful API

## 技术栈

1. 使用 Go 语言
2. 使用 Gin 框架构建 HTTP 服务
3. 使用 GORM 作为 ORM
