# 板块房产关注属性及合法数据获取渠道

> 本文档梳理深圳购房决策中"板块"（商圈/片区）级别需要关注的核心属性，以及各属性合法合规的 API / 数据获取渠道，评估自动化可行性。
>
> 板块定义：市场通用概念，通常指地铁/配套形成的区域认知，如"科技园""碧海""红山""龙华中心"等。粒度介于街道办与小区之间。

## 四、建议方案

### 核心原则

板块数据自动化难点在于：**板块不是法定区划，无统一边界**，主要靠地铁站+市场认知来圈定。建议以地铁站为锚点定义"微板块"，用 POI 密度和通勤时间作为配套指标。

### 自动化管线建议

#### 第一梯队（高优全自动）

1. **高德 POI API**
   - 按板块中心点坐标 + 分类代码批量查询
   - 覆盖：地铁站、学校（1503xx）、医院（0901xx）、商场（0601xx）、公园（1301xx）
   - 建议：每周一次全量轮询，缓存结果

2. **高德路径规划 API**
   - 预设固定通勤方向（如科技园 → 福田中心区）
   - 计算不同板块到主要目的地的耗时/距离
   - 作为板块"通勤价值"量化指标

3. **高德交通态势 API**
   - 采集板块内主要道路的拥堵指数
   - 评估高峰期通勤舒适度

4. **高德/百度行政边界 API**
   - 以街道办边界作为板块地理基础
   - 结合市场认知标注各板块覆盖的街道办范围

#### 第二梯队（半自动，定期更新）

5. **教育局官网学校数据**
   - 爬取各区学校名单、评级
   - 按年度更新（每学期一次）

6. **克尔瑞/贝壳数据订阅**
   - 获取板块均价、成交量等行情数据
   - 按月更新

#### 第三梯队（人工为主）

7. **学区划分映射**
   - 维护 板块/小区 → 学校 的映射表
   - 每年4-5月教育局发布后更新一次

8. **城市规划/旧改**
   - 关注规划局重大公示，人工录入关键进展
   - 建议按季度梳理一次
